# Governance

ASH is maintained by the repository owners under the `dakdevs` GitHub organization/account.

## Decision Making

Maintainers prioritize:

- Correct shell semantics.
- Safe agent execution.
- Small, well-tested Rust modules.
- Plugin APIs that keep the kernel small.
- Compatibility with macOS and Linux first.

Large architectural changes should be proposed in an issue before implementation.

## Maintainer Responsibilities

Maintainers are responsible for:

- Reviewing pull requests.
- Enforcing the Code of Conduct.
- Managing releases.
- Reviewing security reports.
- Keeping release and versioning automation healthy.
